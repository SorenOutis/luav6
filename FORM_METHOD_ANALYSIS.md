# `.form()` Method Research & Definition Summary

## Overview
The `.form()` method is expected to be available on route/action objects throughout the application (e.g., `ProfileController.update.form()`, `store.form()`), but **it is NOT currently defined in the codebase**. This document outlines where it should be defined and how it should work.

---

## 1. Current Usage in Vue Components

### Examples Found:
- **Login.vue**: `v-bind="store.form()"`
- **Profile.vue**: `v-bind="ProfileController.update.form()"`
- **Password.vue**: `v-bind="PasswordController.update.form()"`
- **TwoFactor.vue**: `v-bind="enable.form()"`, `v-bind="disable.form()"`
- **VerifyEmail.vue**: `v-bind="send.form()"`
- **DeleteUser.vue**: `v-bind="ProfileController.destroy.form()"`

All these are used with Inertia.js `<Form>` component: `<Form v-bind="route.method.form()">`

---

## 2. Route/Action Definition Structure

### Generated Route Files:
- Location: `resources/js/routes/` and `resources/js/actions/`
- Generated by: `@laravel/vite-plugin-wayfinder@^0.1.7`
- Example structure:

```typescript
// resources/js/actions/App/Http/Controllers/Settings/ProfileController.ts
export const update = (options?: RouteQueryOptions): RouteDefinition<'patch'> => ({
    url: update.url(options),
    method: 'patch',
})

update.url = (options?: RouteQueryOptions) => {
    return update.definition.url + queryParams(options)
}

update.patch = (options?: RouteQueryOptions): RouteDefinition<'patch'> => ({
    url: update.url(options),
    method: 'patch',
})

const ProfileController = { edit, update, destroy }
export default ProfileController
```

**Note**: The `.form()` method is **NOT** present in these definitions.

## 3. Type Definitions

### Wayfinder Type Definitions:
From `resources/js/wayfinder/index.ts`:

```typescript
export type RouteDefinition<TMethod extends Method | Method[]> = {
    url: string;
} & (TMethod extends Method[] ? { methods: TMethod } : { method: TMethod });

export type RouteFormDefinition<TMethod extends Method> = {
    action: string;
    method: TMethod;
};
```

The `RouteFormDefinition` type exists but is **not being used** to add `.form()` methods to routes.

## 4. Related Technologies

### Inertia.js Integration:
- Using `@inertiajs/vue3@^2.3.18`
- `useForm` hook from Inertia.js is available
- Example: `const form = useForm({ email: '', password: '' })`
- The `<Form>` component accepts form configuration via `v-bind`

### Form Configuration Expected:
When calling `route.method.form()`, it should return form configuration like:
```typescript
{
    action: '/settings/profile',  // The URL to submit to
    method: 'PATCH',              // HTTP method
    // Inertia.js Form props...
}
```

## 5. How `.form()` Should Work

### Expected Implementation Pattern:

```typescript
// This should be added to every route/action object
update.form = () => ({
    action: update.url(),  // Get the URL
    method: 'PATCH',       // HTTP method in uppercase
    // Or could return object compatible with Inertia useForm():
    // { action: string, method: string }
})
```

### Expected Return Type:
```typescript
interface FormProps {
    action: string;  // URL where form submits
    method: string;  // HTTP method (GET, POST, PATCH, DELETE, etc.)
}
```

---

## 6. Where `.form()` Should Be Defined

### Option 1: In Wayfinder Plugin (During Generation)
The `@laravel/vite-plugin-wayfinder` should generate `.form()` methods for each route. Currently, it only generates:
- `.url()`
- `.get()`, `.post()`, `.patch()`, `.delete()`, etc.
- `.definition` property

**Enhancement needed**: Add `.form()` method generation

### Option 2: Via TypeScript Type Augmentation
Create a declaration file to augment route object types:

```typescript
// resources/js/types/routes.d.ts
import { RouteDefinition, RouteFormDefinition } from '@/wayfinder'

declare global {
    interface RouteDefinition {
        form(): RouteFormDefinition;
    }
}
```

### Option 3: Via Runtime Plugin
Extend route objects at runtime with a Vite plugin or app initialization code.

## 8. Implementation Recommendation

### For Code Generation (Recommended):
Modify the Wayfinder plugin to generate:

```typescript
update.form = (): RouteFormDefinition<'patch'> => ({
    action: update.url(),
    method: 'PATCH',
})
```

### For All Route Methods:
Each generated route should have:
- `route()` - returns basic RouteDefinition
- `route.url()` - returns URL string
- `route.METHOD()` - returns RouteDefinition for that method
- `route.form()` - **NEW** - returns FormProps for Inertia Form component

## Summary

The `.form()` method is **missing** from the generated route/action objects. It should:

1. **Return** an object with `action` (URL) and `method` (HTTP method) properties
2. **Be generated** by the Wayfinder plugin or added via type augmentation
3. **Work with** Inertia.js `useForm()` and `<Form>` component
4. **Enable** clean form binding in Vue templates: `v-bind="route.method.form()"`

The method likely needs to be added to the Wayfinder plugin's code generation or implemented as a runtime enhancement to route objects.
